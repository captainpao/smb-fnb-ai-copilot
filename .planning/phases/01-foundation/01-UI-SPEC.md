## Spacing Scale

Fluent UI v9 exposes spacing via CSS custom properties (`--spacingHorizontal*` and `--spacingVertical*`). Map the 8-point scale to Fluent tokens where possible; use raw px only in CSS Modules for layout gaps.

Declared values (must be multiples of 4):

| Token | Value | Fluent Token | Usage |
|-------|-------|--------------|-------|
| xs | 4px | `--spacingHorizontalXS` / `--spacingVerticalXS` | Icon gaps, inline padding, badge internal padding |
| sm | 8px | `--spacingHorizontalS` / `--spacingVerticalS` | Compact element spacing, tab padding |
| md | 16px | `--spacingHorizontalM` / `--spacingVerticalM` | Default element spacing, card internal padding |
| lg | 24px | `--spacingHorizontalL` / `--spacingVerticalL` | Section padding, card-to-card gap |
| xl | 32px | `--spacingHorizontalXL` / `--spacingVerticalXL` | Layout column gaps |
| 2xl | 48px | `--spacingHorizontalXXL` / `--spacingVerticalXXL` | Major section breaks |
| 3xl | 64px | (CSS Modules raw value) | Page-level top/bottom padding on desktop |

Exceptions:
- NavBar touch targets on mobile: minimum 44px height (WCAG 2.5.5 / Apple HIG). Fluent `Tab` height must be at minimum 44px on `<768px` breakpoint via CSS Module override.
- NavBar bottom safe area on mobile: add `padding-bottom: env(safe-area-inset-bottom)` to the fixed bottom nav container for notched devices.

Source: PRD §8 Responsive Design, FEATURES.md §6 Responsive Layout.

## Typography

Fluent UI v9 provides typography via `Title1`, `Title2`, `Body1`, `Caption1`, and `Subtitle2` components backed by design tokens. Use Fluent typography components exclusively — do not create custom `<p>` or `<h*>` tags styled with raw font-size values.

| Role | Fluent Component | Approx Size | Weight | Line Height | Usage |
|------|-----------------|-------------|--------|-------------|-------|
| Body | `Body1` | 14px | 400 (regular) | 1.43 | All body text, descriptions, metadata |
| Label | `Caption1` | 12px | 400 (regular) | 1.33 | Badges, tab labels, supplemental info |
| Heading | `Subtitle2` | 16px | 600 (semibold) | 1.25 | Card headers, section headings |
| Display | `Title2` | 28px | 600 (semibold) | 1.14 | Brand name in NavBar, stub screen headings |

Rules:
- Maximum 4 type roles in use at any time across this phase (enforced above).
- Maximum 2 font weights: 400 (regular) and 600 (semibold).
- Never use inline `style={{ fontSize }}` — always use Fluent typography components or Fluent token CSS variables.
- Brand name "AI Co-Pilot" in NavBar uses `Title2` weight 600.
- Tab labels use `Caption1` weight 400 on mobile; `Body1` weight 400 on desktop.
- Stub screen placeholder text ("Dashboard", "Simulate", "Payments") uses `Title2` weight 600, centered.

Source: PRD §5 Fluent UI Component Mapping, FEATURES.md §6 Responsive Layout.

## Color

All colors are consumed via Fluent UI v9 CSS custom property tokens from `webDarkTheme`. Never use hardcoded hex values in CSS Modules or component styles — always reference `var(--colorToken)`.

| Role | Fluent Token | Approximate Value | Usage |
|------|-------------|-------------------|-------|
| Dominant (60%) | `--colorNeutralBackground1` | #141414 | App shell background, main content area background |
| Secondary (30%) | `--colorNeutralBackground2` | #1a1a1a | NavBar background, card surfaces, sidebar |
| Accent (10%) | `--colorBrandBackground` | Fluent blue #0078d4 approx | Reserved for specific elements listed below |
| Danger/Alert | `--colorPaletteRedBackground3` | Red palette | Warnings, shortfall indicators (Phases 2+) |
| Success | `--colorPaletteGreenBackground3` | Green palette | Positive indicators (Phases 2+) |
| Foreground primary | `--colorNeutralForeground1` | Near-white | All primary text on dark backgrounds |
| Foreground secondary | `--colorNeutralForeground2` | Muted white | Secondary text, labels, placeholders |
| Border | `--colorNeutralStroke1` | Subtle dark border | Card borders, dividers |

Accent (`colorBrandBackground`) is reserved for:
1. Active tab indicator in the NavBar (bottom border or background highlight on the active `Tab`)
2. Primary `Button` appearance background (used in later phases for CTAs; not present in Phase 1 stubs)

Accent is NOT used for: general borders, card headers, all interactive elements, heading text, or decoration.

Phase 1 note: Phase 1 delivers the shell only. Danger/Alert and Success tokens are declared here for continuity but are not rendered in Phase 1 stub screens. They will be first used in Phase 2 (shortfall MessageBar) and Phase 3 (priority badges).

Source: PRD §1a Design System — Theme section, additional_context block (locked decisions).

## Layout Contract

### App Shell Container

```css
/* App.module.css */
.appShell {
  min-height: 100vh;
  background-color: var(--colorNeutralBackground1); /* #141414 */
  display: flex;
  flex-direction: column;
}
```

### Main Content Area

```css
.mainContent {
  flex: 1;
  max-width: 1200px;
  width: 100%;
  margin: 0 auto;
  padding: var(--spacingVerticalL) var(--spacingHorizontalL); /* 24px */
}
```

### NavBar — Mobile (< 768px): Fixed Bottom

```css
.navBar {
  position: fixed;
  bottom: 0;
  left: 0;
  right: 0;
  background-color: var(--colorNeutralBackground2); /* #1a1a1a */
  padding-bottom: env(safe-area-inset-bottom);
  border-top: 1px solid var(--colorNeutralStroke1);
  z-index: 100;
}

/* Reserve space so content does not scroll behind the fixed nav */
.mainContent {
  padding-bottom: calc(60px + env(safe-area-inset-bottom));
}
```

### NavBar — Desktop (>= 768px): Fixed Top

```css
@media (min-width: 768px) {
  .navBar {
    position: sticky;
    top: 0;
    bottom: auto;
    border-top: none;
    border-bottom: 1px solid var(--colorNeutralStroke1);
    padding-bottom: 0;
  }

  .mainContent {
    padding-bottom: var(--spacingVerticalL);
  }
}
```

### Responsive Breakpoints

| Breakpoint | Value | Layout change |
|------------|-------|---------------|
| Mobile | < 768px | Single column; NavBar fixed at bottom |
| Tablet | 768px – 1024px | Two-column layout (Phase 2+); NavBar at top |
| Desktop | > 1024px | Three-column layout (Phase 2+); NavBar at top |

Source: PRD §8 Responsive Design, REQUIREMENTS.md RESP-01/02/03, additional_context block.

## NavBar Specification

### Structure

```
[ Brand: "AI Co-Pilot" ]    [ Dashboard | Simulate | Payments ]
```

- Mobile: brand name hidden or abbreviated to save space; tabs span full width; fixed at bottom.
- Desktop: brand name left-aligned; tabs right-aligned or centered; sticky at top.

### Tab Labels

| Tab | Label | Route | Icon (Phase 1: optional) |
|-----|-------|-------|--------------------------|
| 1 | Dashboard | `/` (HashRouter: `#/`) | `HomeRegular` / `HomeFilled` from @fluentui/react-icons |
| 2 | Simulate | `/simulate` | `DataTrendingRegular` / `DataTrendingFilled` |
| 3 | Payments | `/payments` | `PaymentRegular` / `WalletRegular` |

Active tab: indicated by Fluent `Tab` selected state, which applies `colorBrandBackground` underline by default in `webDarkTheme`. No custom active styling needed.

### Active State

Use `selectedValue` on `TabList` driven by `useLocation()` from `react-router-dom`. On tab click, use `useNavigate()` to push the route. Do not use `<Link>` inside `Tab` — navigate imperatively via `onTabSelect`.

Source: PRD §8 Routing, REQUIREMENTS.md NAV-01/02/03.

## Interaction Contract

| Interaction | Trigger | Response | State change |
|------------|---------|----------|--------------|
| Tab press (Dashboard) | Click / tap on "Dashboard" tab | Navigate to `#/`; tab visual active state updates | None (navigation only) |
| Tab press (Simulate) | Click / tap on "Simulate" tab | Navigate to `#/simulate`; tab visual active state updates | None (navigation only) |
| Tab press (Payments) | Click / tap on "Payments" tab | Navigate to `#/payments`; tab visual active state updates | None (navigation only) |
| Direct URL navigation | Browser back/forward or direct hash URL | Correct tab shows as active; correct stub screen renders | None |
| Resize below 768px | Viewport resize | NavBar moves to fixed bottom position | CSS-driven only, no JS |
| Resize above 768px | Viewport resize | NavBar moves to sticky top position | CSS-driven only, no JS |

Keyboard: Fluent `TabList` handles keyboard navigation (`Arrow Left/Right`, `Home`, `End`) natively. No custom keyboard handler needed.

Focus management: Tab focus ring uses Fluent's default `:focus-visible` styling (blue ring). Do not suppress or override focus rings.

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| WCAG AA contrast on all text | Satisfied by `webDarkTheme` tokens — `colorNeutralForeground1` on `colorNeutralBackground1` passes 7:1. Never use custom hex that bypasses this. |
| NavBar landmark | Wrap `TabList` in `<nav aria-label="Main navigation">` |
| Tab selection announced | Fluent `TabList` + `Tab` uses `role="tablist"` / `role="tab"` + `aria-selected` natively |
| Touch target minimum | NavBar tab touch targets: minimum 44px height on mobile |
| No horizontal scroll | `overflow-x: hidden` on `body` for mobile; single-column layout prevents overflow |
| Scrollbar styling | Custom scrollbar CSS is cosmetic only; keyboard scrolling unaffected |
